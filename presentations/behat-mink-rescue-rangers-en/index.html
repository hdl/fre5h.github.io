A PHP framework for testing your business expectations

Konstantin Kudryashov

Creator of Behat and Mink
You might think about Behat as a tool to teach your clients about testing, but in reality it's a tool to teach you about client business.
  • Behat has been developed under the influence of Cucumber
  • PHP-framework (>=PHP 5.3) for a testing of behavior of application
  • Gherkin DSL - easy syntax for writing stories
  • Comfortable writing tests using annotations
  • Testing of anything, including interaction with the browser using Mink

Install

  • Composer
  • PHAR
  • Git

Composer

  1. Create file composer.json in the project directory
    {
    "require": {
        "behat/behat": "2.4.*@stable"
        },
    "minimum-stability": "dev",
    "config": {
        "bin-dir": "bin/"
    }
}
  2. Download composer.phar and run command install 
    $ curl http://getcomposer.org/installer | php
$ php composer.phar install
  3. And then you can use it 
    $ bin/behat

PHAR

$ wget https://github.com/downloads/Behat/Behat/behat.phar
$ php behat.phar

Git

  1. Клонуємо з GitHub
    $ git clone git://github.com/Behat/Behat.git && cd Behat
$ git submodule update –init
  2. Завантажуємо composer.phar і запускаємо з командою install 
    $ wget -nc http://getcomposer.org/composer.phar
$ php composer.phar install
  3. І можемо користуватись
    $ bin/behat

Initialization

  1. Execute command
    $ behat --init
  2. Structure of directories
    |-- features
   `-- bootstrap
      `-- FeatureContext.php
    • features/ - Files *.feature, that Behat will be executing
    • features/bootstrap/ - All files *.php from this directory will be included automatically when you run Behat
    • features/bootstrap/FeatureContext.php - Here will be stored all definitions of your steps

Configuration

  • By default configuration is stored in behat.yml
  • If necessary, you can run a different configuration:
    $ behat --config custom-config.yml
  • Example of configuration: 
    default:
    paths:
        features: features
        bootstrap: %behat.paths.features%/bootstrap

check_errors:
    filters:
        tags: "@error"
    formatter:
        name: progress

ci:
    formatter:
        name:       junit
        parameters:
            output_path: /var/tmp/junit
  • Run with different profiles:
    $ behat --profile check_errors
$ behat --profile ci

Gherkin DSL

Feature: Describe some feature of application
    In order to realize a named business value
    As a some user
    I need to be able to do some action

    Background:
        Given prepare system to known stage

    Scenario: Check some business logic of application
        Given some precondition
          And some other precondition
         When some action by user
          And some other action
         Then some testable outcome is achieved
          And something else we can check happens too
          But something should not happen

/** @Given /^there is Morpheus$/ */
public function thereIsMorpheus() { throw new PendingException(); }
/** @Given /^there is Neo$/ */
public function thereIsNeo() { throw new PendingException(); }
/** @Given /^there is a blue pill$/ */
public function thereIsABluePill() { throw new PendingException(); }
/** @Given /^there is a red pill$/ */
public function thereIsARedPill() { throw new PendingException(); }
/** @When /^Neo eat red pill$/ */
public function neoEatRedPill() { throw new PendingException(); }
/** @Then /^he should stay in Wonderland and see how deep the rabbit-hole goes$/ */
public function heShouldStayInWonderlandAndSeeHowDeepTheRabbitHoleGoes() { throw new PendingException(); }
/** @Given /^the story should not end$/ */
public function theStoryShouldNotEnd() { throw new PendingException(); }

Every step is a method

/**
 * Feature context
 */
class FeatureContext extends MinkContext
{
    /**
     * Go to the homepage and check that it is enabled
     * and returns status code 200
     *
     * @Given /^I am on the homepage$/
     */
    public function iAmOnHomepage()
    {
        $this->visit('/');
        $this->assertResponseStatus(200);
    }
}

@Given, @When, @Then

  • Steps are put to FeatureContext.php
  • @And, @But - helpers for easier sentences
  • Behat doesn't see the difference between all kind of steps, but you should
  • If step didn't throw an exception - it is passed successfully
  • Use assert-methods from PHPUnit

Runing tests from console

  • Run all tests: 
    $ behat
  • Run some feature: 
    $ behat features/some.feature
    or 
    $ behat --name="Feature name"
  • Run some scenario from some feature: 
    $ behat features/some.feature:15

Hooks

  • @BeforeSuite, @AfterSuite - executing for all suites of features
  • @BeforeFeature, @AfterFeature - executing for each feature
  • @BeforeScenario, @AfterScenario - executing for each scenario
  • @BeforeStep, @AfterStep - executing for each step
/** @BeforeFeature */
public static function setupFeature(FeatureEvent $event)
{
    // prepare something
}

/** @AfterFeature */
public static function teardownFeature(FeatureEvent $event)
{
    // clean something
}

Tags

  • @blog
    Feature: Blog
        ...

    @login
    Scenario: User login
        ...
  • You can run only some scenarios: 
    $ behat --tags '@login,@logout'
$ behat --tags 'login,error'
$ behat --tags '@login&&@admin'
$ behat --tags '~@error'
  • You can customize hooks for only some scenarios: 
    /**
 * Load fixtures for Voucher feature
 *
 * @BeforeScenario @voucher-feature
 */
public function loadFixturesForVoucherFeature()
{
    $this->loadFixtures('Path\To\Fixtures\VoucherFeature');
}

Acceptance test framework for web applications

  • PHP-framework (>=PHP 5.3) for testing web-applications
  • Gives API to use different browser emulators
  • Has API for the next emulators:
    • Goutte
    • Zombie.js
    • Selenium
    • Selenium2
    • Sahi
